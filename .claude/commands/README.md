## 📋 利用可能なコマンド

### `/commit-push-pr` - Git操作一括実行

**目的**: コミット、プッシュ、PR作成を一連の流れで実行

**使用方法**:
```
/commit-push-pr
```

**実行内容**:
1. ✅ 現在の変更状況を確認 (git status, diff, log)
2. ✅ コミットメッセージの提案とユーザー確認
3. ✅ ファイルのステージング (git add -A)
4. ✅ コミット作成 (適切なメッセージ付き)
5. ✅ リモートへプッシュ
6. ✅ PR作成 (オプション)

**特徴**:
- 🎯 インタラクティブな確認プロセス
- 🚀 並列実行による高速処理
- 🔒 セキュリティチェック (.envファイル等)
- 📝 Claude Codeフッター自動追加
- 🐛 エラーハンドリングとリトライ機能

**推奨される使用シーン**:
- 機能開発完了時
- バグ修正完了時
- ドキュメント更新時
- Phase完了時のマイルストーン

**例**:
```bash
# 1. 変更を加える
vim backend/src/server.js

# 2. コマンド実行
/commit-push-pr

# 3. 対話的に確認しながら進行
# - コミットメッセージ確認
# - PR作成の有無選択
# - ベースブランチ選択

# 4. 完了!
```

## 🚨 注意事項

1. **コミット前の確認**:
   - 意図しないファイルが含まれていないか
   - .envファイルや秘密情報が含まれていないか
   - 大きなバイナリファイルがないか

2. **PR作成時**:
   - ベースブランチが正しいか確認
   - CI/CDパイプラインの準備ができているか
   - レビュアーが利用可能か

3. **エラー発生時**:
   - エラーメッセージを注意深く読む
   - pre-commit hookによる自動修正を確認
   - 必要に応じて手動で修正
